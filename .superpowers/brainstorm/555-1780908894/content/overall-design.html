Claude Monitor 整体设计方案

所有已确认设计的汇总

一、技术栈

后端
  • JDK 21 + SpringBoot 3.0
  • SQLite（嵌入式数据库）
  • JSONL 文件增量解析器
  • Hooks 事件接收端点
  • REST API 供前端调用
前端（Tauri 桌面应用）
  • Tauri 2.x（Rust 核心 + WebView）
  • 前端框架：React + TypeScript
  • 动画：Lottie（轻量矢量动画）
  • 图表：ECharts / Chart.js

二、系统架构

JSONL 文件
定时扫描 5s
Hooks 事件
实时推送
↓
SpringBoot 后端
数据采集 → 解析 → 存储 → API
SQLite 嵌入式数据库
↓ REST API + WebSocket
悬浮窗
折叠/展开 + Lottie 动画
WebSocket 实时状态
Web 仪表盘
用量/费用/趋势/历史
REST API 查询

三、功能模块

P0 核心功能 Token 用量统计 per-session / per-day / per-month 的 token 消耗与费用估算 实时状态悬浮窗 折叠态动画图标 + 悬停展开数据面板，4 种状态动画 JSONL 解析器 增量扫描 .claude/projects 目录，提取 token / 工具 / 会话数据 Hooks 接收端 接收 PostToolUse / UserPromptSubmit / Stop 事件，驱动实时状态
P1 增强功能 Web 仪表盘 概览卡片 + 趋势图 + 工具分布 + 会话列表 形象系统 内置熊猫/猫咪/机器人，Lottie 动画，支持自定义形象包 工具调用统计 各工具使用频率、耗时、参数分析
P2 未来功能 会话历史浏览 对话内容查看、搜索、回放 预算告警 达到预算阈值时通知 多项目对比 跨项目的用量对比分析

四、数据模型

Session（会话）
id, project, model, startedAt, endedAt,
totalInputTokens, totalOutputTokens,
totalCacheTokens, estimatedCost,
toolCallCount, status
ToolCall（工具调用）
id, sessionId, toolName, inputParams,
timestamp, duration
DailyUsage（日用量）
date, totalInputTokens, totalOutputTokens,
totalCacheTokens, estimatedCost,
sessionCount, toolCallCount
AgentStatus（实时状态）
sessionId, status (working/done/waiting/idle),
currentTool, lastUpdatedAt
