设计详情：数据采集层

如何从 Claude Code 获取监控数据

数据流架构

Claude Code

→
数据采集服务
SpringBoot 后端
→
数据库
SQLite / H2
→
REST API
供前端调用

采集方式一：JSONL 文件解析器

扫描目录：C:\Users\DELL\.claude\projects\

解析策略：

  • 定时扫描（每 5s）检测新会话文件
  • 增量解析：记录每文件的已读取行号，只读新增部分
  • 过滤 token=0 的中间消息，只保留最终 assistant 消息的 usage

提取数据：

  • Token 用量：input_tokens, output_tokens, cache tokens
  • 工具调用：tool_use 类型的 name + input
  • 会话元信息：model, gitBranch, timestamp, version
  • 对话内容：user prompt + assistant text（可选存储）

采集方式二：Hooks 实时推送

注册 Hook 脚本：在 settings.json 中配置 Hook

Hook 脚本功能：将事件数据 POST 到 SpringBoot 后端的 /api/hooks/event

捕获的事件：

  • PostToolUse → 工具执行完成，更新实时状态
  • UserPromptSubmit → 用户提交新 prompt，标记"工作中"
  • Stop → Agent 停止，标记"完成"
  • Notification → 权限请求等运维事件

实时状态映射：

  • UserPromptSubmit → 🔴 工作中（动画：熊猫敲键盘）
  • PostToolUse → 🟡 执行工具（动画：熊猫操作中）
  • Stop → 🟢 完成（动画：熊猫举 DONE 牌）
  • Notification(permission) → 🟠 等待确认（动画：熊猫等待）
